## Known Limitations (v1)
•	One fixed 3.5 km adjacency rule for every job type, even though Full Time and On Demand helpers really have different commute tolerances.
•	Apartment search works by name only. If two buildings share a name across localities, this will need fixing at a larger scale.
•	No usage tracking yet, so we can't yet see which features agents actually use most.
•	Only 15 localities and 20 apartments are loaded right now. The real dataset is much bigger, so this is a working prototype, not the final production tool.

## Suggested Next Features (v2)
•	Different adjacency distances for different job types, instead of one fixed number for everyone.
•	Smarter apartment search that also shows the locality, so duplicate building names don't cause confusion.
•	Basic usage tracking, so we can see which features agents rely on most.
•	A simple coverage view for Sourcing Team Leads, using the same map and data.
•	Sign-in restricted to EzyHelpers email addresses, if this becomes a permanent internal tool.
