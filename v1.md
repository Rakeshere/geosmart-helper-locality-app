## Known Limitations (v1)

- Fixed **3.5 km** proximity rule for all job types.
- Apartment search supports **name only**; duplicate names may cause confusion.
- No usage analytics or user activity tracking.
- Prototype dataset includes only **15 localities** and **20 apartments**.
- Transit data depends on the **Overpass API**, so availability may vary.

---

## Future Improvements (v2)

- Support **dynamic proximity distances** based on job type.
- Improve apartment search by displaying **apartment + locality**.
- Add **usage analytics** to understand feature adoption.
- Build a **coverage dashboard** for Sourcing Team Leads.
- Restrict access using **EzyHelpers email authentication**.
- Scale to **hundreds of localities** and **thousands of apartments**.
- Add advanced filters (Zone, Pincode, Transit, Apartment).
- Enable **real-time database updates** instead of static seed data.
